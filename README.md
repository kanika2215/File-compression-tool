# Huffman File Compression Tool

A C++ implementation of the Huffman Coding algorithm for lossless file compression and decompression. This project demonstrates the practical use of data structures such as binary trees, priority queues, hash maps, and file handling to reduce file size while preserving all original information.

The application supports compressing any file type, inspecting compressed files, and restoring the original data through decompression.

---

## Features

### File Compression
- Compresses files using Huffman Coding
- Supports text and binary files
- Generates compact `.huff` compressed files
- Displays compression statistics and space savings

### File Decompression
- Restores the original file exactly
- Reconstructs the Huffman tree automatically
- Handles compressed files generated by the program

### File Inspection
- View metadata stored inside a `.huff` file
- Original file size
- Compressed file size
- Tree size
- Compression ratio
- Padding information

### Testing Utility
- Automatically generate a sample text file
- Useful for testing compression performance

### User-Friendly Interface
- Interactive menu-driven CLI
- Colored terminal output
- Command-line mode for automation

## Data Structures Used

### Huffman Tree
Stores characters and their frequencies in a binary tree structure.

```cpp
struct HuffNode {
    unsigned char ch;
    int freq;
    HuffNode* left;
    HuffNode* right;
};
```

### Priority Queue

Used to repeatedly select the two least frequent nodes while building the Huffman tree.

```cpp
priority_queue<HuffNode*, vector<HuffNode*>, NodeCmp>
```

### Frequency Table

Stores character frequencies.

```cpp
unordered_map<unsigned char, int>
```

## How Huffman Coding Works

1. Read the input file.
2. Count the frequency of each character.
3. Build a Huffman Tree using a min-heap.
4. Generate binary codes for every character.
5. Replace characters with their codes.
6. Store:
   - Original file size
   - Serialized Huffman tree
   - Compressed bit stream
7. Save everything into a `.huff` file.

For decompression:

1. Read the compressed file.
2. Reconstruct the Huffman Tree.
3. Decode the bit stream.
4. Restore the original file.

Since Huffman Coding is lossless, the decompressed file is identical to the original.

## Compressed File Format

The `.huff` file contains:

```text
[4 bytes] Original file size
[4 bytes] Huffman tree bit length
[Tree Data]
[1 byte ] Padding count
[Compressed Data]
```

This information allows the original file to be reconstructed during decompression.

---

### Command-Line Compression

```bash
./huffman compress input.txt output.huff
```

Example:

```bash
./huffman compress sample.txt sample.huff
```

---

### Command-Line Decompression

```bash
./huffman decompress sample.huff restored.txt
```

Example:

```bash
./huffman decompress sample.huff recovered.txt
```

## Algorithm Complexity

### Compression

| Operation | Complexity |
|------------|------------|
| Frequency Counting | O(n) |
| Tree Construction | O(k log k) |
| Encoding | O(n) |

### Decompression

| Operation | Complexity |
|------------|------------|
| Tree Reconstruction | O(k) |
| Decoding | O(n) |

Where:

- `n` = number of bytes in the file
- `k` = number of unique characters
